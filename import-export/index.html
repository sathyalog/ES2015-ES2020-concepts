Import & Export
The static import statement is used to import bindings which are exported by another module.
The export statement is used when creating JavaScript modules to export functions, objects, or primitive values from the module so they can be used by other programs with the import statement.

Import & Export: 

Types of Imports:

    1.Import an entire module's contents.

        Eg:import * as myModule from '/modules/my-module.js';

    2.Import a single export from a module

        Eg:import {myExport} from '/modules/my-module.js';

    3.Import multiple exports from module

        Eg:import {foo, bar} from '/modules/my-module.js';

    4.Import an export with a more convenient alias

        Eg:import {reallyReallyLongModuleExportName as shortName} from '/modules/my-module.js';

    5.Import a module for its side effects only

        Eg:import '/modules/my-module.js';

    6. Dynamic Imports

        Eg: import('/modules/my-module.js')
                .then((module) => {
                    // Do something with the module.
            });
Types of exports:

    1.Named Exports (Zero or more exports per module)

    2.Default Exports (One per module)

The import statement cannot be used in embedded scripts unless such script has a type="module".
